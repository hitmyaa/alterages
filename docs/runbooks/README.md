# Runbooks

Procédures opérationnelles pour les actions courantes et la gestion d'incidents.

## Index (à compléter)

- `deploiement-staging.md` — Promotion d'une branche vers staging
- `deploiement-production.md` — Promotion vers production
- `rollback.md` — Revenir en arrière en cas de régression
- `incident-database.md` — Procédure en cas d'incident base de données
- `incident-paiement.md` — Procédure en cas d'incident paiement
- `ajout-migration.md` — Créer et tester une nouvelle migration Supabase

## Principes

- Chaque runbook doit être **actionnable à froid**, sans contexte préalable.
- Préférer des étapes numérotées avec commandes exactes copiables-collables.
- Toujours préciser qui peut exécuter la procédure (droits requis).
